# Чек-лист приёмки MVP

## Инфраструктура

- [ ] `docker compose up --build` поднимает все сервисы без ошибок
- [ ] Все контейнеры в состоянии `running`
- [ ] После перезапуска данные PostgreSQL и MinIO сохраняются
- [ ] Миграции применяются автоматически или по команде
- [ ] `.env.example` содержит все необходимые переменные с комментариями

## Аутентификация

- [ ] Вход через Discord OAuth создаёт/обновляет пользователя и привязку
- [ ] Вход через логин/пароль работает
- [ ] Привязка Discord по snowflake сохраняется
- [ ] Без логина доступна только страница входа
- [ ] Защищённые маршруты перенаправляют на логин

## Workspace и гильдии

- [ ] Выбор workspace работает (если несколько)
- [ ] Список гильдий соответствует серверам, куда добавлен бот
- [ ] Настройка канала для consent сохраняется
- [ ] После отзыва бота гильдия помечается или скрывается

## Запись через Discord бота

- [ ] Бот подключается к голосовому каналу по команде
- [ ] В БД появляются `participant_intervals` при входе/выходе участников
- [ ] Файлы дорожек появляются в MinIO
- [ ] Сообщение о записи видно в канале или в ЛС
- [ ] Кнопка «Остановить запись» завершает запись и обновляет статус

## Очереди и обработка

- [ ] Завершение записи приводит к постановке задачи VAD
- [ ] Воркер обрабатывает задачи по очереди
- [ ] Повторная постановка той же задачи не создаёт дубликатов данных
- [ ] При падении воркера задачи остаются в Redis и обрабатываются после рестарта

## Транскрипция

- [ ] После успешной транскрипции в БД есть transcript и сегменты с таймкодами
- [ ] Сегменты привязаны к discord_user_id
- [ ] Таймкоды соответствуют длительности записи

## Саммари

- [ ] Создание саммари по записи сохраняет result_json
- [ ] Структура соответствует схеме (title, summary, decisions, action_items, risks, topics)
- [ ] Для пользователя с частичным доступом в ответе только допустимые evidence
- [ ] При невалидном JSON задача в failed с причиной

## Поиск

- [ ] После индексации поиск по фразе из транскрипта возвращает релевантный чанк
- [ ] Семантический запрос (перефраз) возвращает подходящие результаты
- [ ] Пользователь с частичным доступом не видит результаты из недоступных интервалов
- [ ] Админ видит по всем записям

## ACL на половинную запись

- [ ] Запрос транскрипта от пользователя, бывшего только первые 10 минут, возвращает только сегменты в этих 10 минутах
- [ ] Прямой запрос медиа на 20–30 мин для этого пользователя — 403
- [ ] В UI пользователь видит только свои интервалы
- [ ] Попытка обхода через API с подставленным recording_id и диапазоном вне allowed_ranges — 403

## UI и плеер

- [ ] Открытие записи показывает плеер и транскрипт
- [ ] Клик по сегменту перематывает плеер
- [ ] Воспроизведение работает в разрешённом диапазоне
- [ ] При отсутствии доступа к записи показывается 404 или редирект

## Формат API

- [ ] Неавторизованный запрос к `/api/v1/recordings` возвращает 401
- [ ] Запрос записи, к которой нет доступа — 404 или 403
- [ ] Ответ одной записи содержит все обязательные поля формата ТЗ
- [ ] Медиа URL истекает через TTL

## Известные ограничения

- VAD пока не реализован (заглушка)
- Главы не реализованы
- Редактор транскрипта не реализован
- RAG-чат не реализован
- Экспорт не реализован
- Redaction не реализован
- Шэринг не реализован
- Интеграции (WEEEK, webhook) не реализованы
